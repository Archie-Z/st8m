欢迎来到 **ST8-M**——一个由 Archie 构建与维护的数字化思维与工程日志系统。ST8 是 **State** 的简写，而 **M** 则承载了多重含义：它是记录状态的 **Machine**，是组织系统的 **Manager**，亦是探究运作原理的 **Mechanics**。

这里是思考、构建与演化的存档。

### 关于文记 (The Notes)

文记是 ST8-M 系统中的主要内容载体，主要收录技术解析文章、工程实践日志，以及游戏设计笔记与跨领域思考。

#### 状态索引 (State Index)

内容遵循 `[Type].[Number]-[Title]` 的命名规则。这不仅是一种秩序，也是对“万物皆状态，万物可系统化”这一核心思想的实践。

| 状态类型码 | 全称与描述 |
| :--- | :--- |
| **`F`** | **Frontend** - 前端架构、交互工程与现代 Web 技术栈探索。 |
| **`B`** | **Backend** - 服务端逻辑、API 设计、数据库与分布式系统思考。 |
| **`S`** | **System** - 系统设计、工具链、工作流与底层原理剖析。 |
| **`G`** | **Game** - 游戏机制设计、图形编程、引擎原理与开发日志。 |
| **`P`** | **Prototype** - 原型设计、快速验证、创意技术与项目孵化过程。 |
| **`X`** | **Testing** - 测试策略、自动化、质量保证与可观测性实践。 |
| **`O`** | **Operations** - 运维、部署、基础设施即代码与云原生技术。 |
| **`U`** | **Linux/Unix** - 系统哲学、命令行艺术、内核与开源生态。 |
| **`D`** | **System Design** - 复杂系统抽象、架构范式与解决方案设计。 |
| **`T`** | **Thought** - 跨领域思维、逻辑推演、设计哲学与认知碎片。 |
| **`L`** | **Log** - 周期记录、项目复盘、学习轨迹与阶段性总结。 |

### 关于随笔 (The Jotting)

随笔是 ST8-M 系统中的一个更轻量级的记录空间，用于捕捉和分享那些零散的思绪、瞬间的灵感、日常的观察或简短的心得体会。

#### 状态索引 (State Index)

随笔不进行内容上的分类，仅仅使用`E.[number]-[Title]`进行序列化编码，通过Tag进行标识。

### 技术栈 (The Stack)

-   **生成引擎**：[Astro](https://astro.build) - 追求内容速度与纯粹的 Web。
-   **主题框架**：[ThoughtLite](https://github.com/tuyuritio/astro-theme-thought-lite) - 简洁、可读的博客主题。
-   **部署平台**：[Vercel](https://vercel.com) - 无缝的全球部署体验。
-   **存储与协同**：[GitHub](https://github.com) - 所有内容与源码的版本库。
